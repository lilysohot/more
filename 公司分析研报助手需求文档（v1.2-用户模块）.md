## 一、项目概述

### 1.1 项目背景

当前公司分析流程已形成成熟的"三维合一投资决策委员会框架"，包含芒格视角、产业专家视角、审计专家视角三个维度的深度分析。为提高分析效率、降低使用门槛，需要开发一个网页版工具，用户只需输入公司名称，即可自动生成专业的投资分析研报。

### 1.2 项目目标

- **简化操作**：用户只需输入公司名称，一键生成研报
- **标准化输出**：统一研报格式，确保分析质量
- **多格式支持**：同时生成 Markdown 和 HTML 两种格式
- **可视化展示**：HTML 版本包含数据图表，直观呈现分析结果
- **用户友好**：支持用户注册登录，可自定义配置 LLM API

### 1.3 目标用户

- 个人投资者
- 投资分析师
- 基金研究员
- 财经媒体从业者

### 1.4 Skill 框架

本项目基于 Skill 分析框架实现自动化分析，核心 Skill 包括：

| Skill | 类型 | 作用 | 文档位置 |
|-------|------|------|----------|
| investment-analyst | 提示词 | 三维合一投资分析框架 | `.trae/skills/investment-analyst/SKILL.md` |
| analyzing-financial-statements | Python 算法 | 财务比率计算与解释 | `.trae/skills/analyzing-financial-statements/` |
| company-analysis-workflow | 提示词 | 标准化分析工作流程 | `.agents/skills/company-analysis-workflow/SKILL.md` |
| data-visualization | 提示词 | 数据可视化指导 | `.agents/skills/data-visualization/SKILL.md` |

> 📚 **详细说明**：各 Skill 的使用方法和调用流程详见《Skill 集成说明文档》

## 三、用户模块设计

### 3.1 用户注册与登录

#### 注册功能
- **注册方式**：
  - 邮箱注册（发送验证码）
  - 手机号注册（发送短信验证码）
- **注册信息**：
  - 用户名
  - 邮箱/手机号
  - 密码（加密存储）
  - 验证码
- **验证流程**：
  - 输入验证码
  - 验证通过 → 创建账户
  - 验证失败 → 提示错误

#### 登录功能
- **登录方式**：
  - 账号密码登录
  - 验证码登录（免密码）
- **会话管理**：
  - Token 认证（JWT）
  - 会话过期时间：7 天
  - 支持"记住我"功能（30 天）

### 3.2 个人中心

#### 个人信息
- 用户名
- 邮箱/手机号
- 注册时间
- 使用统计（分析报告数量）

#### 使用记录
- 历史分析报告列表
- 报告生成时间
- 快速访问已保存的报告

### 3.3 API 配置管理

#### 预置 LLM 模型

系统自带 4 种 LLM 模型供用户选择：

| 模型名称 | 提供商 | 说明 | 配置难度 |
|---------|--------|------|---------|
| **DashScope（通义千问）** | 阿里云 | 国产大模型，性价比高 | ⭐ 简单 |
| **Claude** | Anthropic | 强大的推理能力 | ⭐⭐ 中等 |
| **OpenAI GPT-4** | OpenAI | 行业领先 | ⭐⭐⭐ 较复杂 |
| **本地部署模型** | 自建 | 数据隐私保护 | ⭐⭐⭐⭐ 复杂 |

#### 自定义模型配置

用户可以添加自己的 LLM API 配置：

**配置项**：
- **模型名称**：自定义显示名称（如"我的 GPT-4"）
- **API 提供商**：选择或自定义
- **API Key**：用户自己的 API 密钥
- **Base URL**：API 端点地址（可选，默认使用预置）
- **模型版本**：选择具体模型版本（如 gpt-4-turbo）
- **是否默认**：设置为默认使用的模型

**配置流程**：
```
用户进入 API 配置页面
    ↓
点击"添加新模型"
    ↓
填写配置信息
    ↓
测试连接（可选）
    ↓
保存配置
    ↓
设置为默认模型（可选）
```

**配置示例**：
```json
{
  "model_name": "我的 GPT-4",
  "provider": "openai",
  "api_key": "sk-xxxxxxxxxxx",
  "base_url": "https://api.openai.com/v1",
  "model_version": "gpt-4-turbo-preview",
  "is_default": true
}
```

**安全要求**：
- API Key 加密存储
- 不显示完整密钥（脱敏展示：`sk-****xxxx`）
- 支持删除和修改配置
- 每个用户可配置多个模型（上限 10 个）

## 五、报告输出规范

### 5.1 报告结构

```markdown
# [公司名称]（股票代码）深度分析报告

**报告日期**：YYYY 年 MM 月 DD 日
**分析师**：三维合一顶级投资分析师

---

## 第一步：身份穿透与定性

### 公司本质
> [一句话定义公司本质]

**一句话定性**：[详细定性描述]

### 核心财务数据概览
| 指标 | 数值 | 同比变化 | 数据来源 | 报告日期 |
|------|------|----------|----------|----------|
| ... | ... | ... | ... | ... |

---

## 第二步：三维深度辩论

### 一、【芒格视角】评估护城河、管理层诚信、估值安全边际
### 二、【产业专家视角】拆解物理瓶颈、供需缺口、成本曲线位置
### 三、【审计专家视角】排查关联交易、资金流向、资产质量

---

## 第三步：综合评级与策略

### 最终得分：X 分（满分 10 分）
### 芒格的决定：[买入/卖出/持有/太难了]
### 核心理由：[...]

---

## 风险提示
## 数据来源汇总
```

### 5.2 HTML 报告要求

| 要求项 | 说明 |
|--------|------|
| 响应式布局 | 支持 PC 端和移动端浏览 |
| 图表可视化 | 使用 Chart.js 等库展示数据图表 |
| 样式统一 | 统一的配色方案和排版风格 |
| 图表整齐 | 图表排列整齐，禁止错位和乱序 |
| 数据标注 | 每个数据标注精确来源和日期 |

### 5.3 必备图表清单

| 图表名称 | 图表类型 | 说明 |
|----------|----------|------|
| 核心财务数据 | 表格 | 营收、净利润、毛利率等 |
| 业务收入占比 | 饼图/环形图 | 各业务板块收入占比 |
| 毛利率对比 | 柱状图 | 各业务毛利率对比 |
| 净利润趋势 | 折线图/柱状图 | 近 3 年净利润变化 |
| ROE 趋势 | 折线图 | 净资产收益率变化 |
| 资产负债率对比 | 柱状图 | 公司 vs 行业平均 |
| 现金流 vs 净利润 | 柱状图 | 盈利质量分析 |
| 综合评分雷达图 | 雷达图 | 五维度评分展示 |

## 十一、风险与应对

| 风险项 | 风险等级 | 应对措施 |
|--------|----------|----------|
| 数据源不稳定 | 高 | 多数据源备份，缓存机制 |
| LLM API 限流 | 中 | 请求队列，重试机制 |
| 分析结果不准确 | 中 | 人工审核机制，用户反馈 |
| 爬虫被封禁 | 中 | 代理池，请求频率控制 |
| 用户数据泄露 | 高 | 加密存储，权限控制 |
| API 密钥泄露 | 高 | 加密存储，脱敏展示 |
