# API 模块规则

适用目录：`backend/app/api/`

必读：`../../docs/api-patterns.md`

## 关键规则

- 路由文件按资源拆分，新增资源后同步注册到 `backend/app/main.py`。
- Handler 保持薄层：收参、依赖、调用 service、组装响应，不堆业务流程。
- 受保护接口默认依赖 `get_current_user`，数据库会话统一使用 `Depends(get_db)`。
- 新接口优先使用 `UUID`、`Query(...)` 等强类型校验；不要继续复制 `str -> UUID` 的存量写法。
- 稳定接口必须声明 `response_model`。创建接口返回 `201`，删除接口返回 `204`。
- 路由必须按用户归属过滤数据，不能只按资源主键读取。
- 涉及密钥的响应只能返回掩码值，不能返回明文。
- 静态路径先于动态路径声明，避免 `/{id}` 抢占匹配。

## 不要做的事

- 不要在这里手动创建数据库 session。
- 不要把外部网络错误、内部异常细节直接透传给前端。
- 不要无测试地改动前端已消费的响应结构。
