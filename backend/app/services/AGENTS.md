# Service 模块规则

适用目录：`backend/app/services/`

必读：`../../docs/database-rules.md`

## 关键规则

- 可复用的数据库读写流程、跨表更新、默认值切换、状态推进放在 service 层。
- 公开写方法负责事务收口，尽量只在边界处 `commit()` 一次。
- 内部 helper 如果只是更大流程的一步，不要再单独 `commit()`；`_unset_default()` 是存量实现，不作为新代码模板。
- 需要返回数据库最终值时，`commit()` 后再 `refresh()`。
- 计数使用 `func.count(...)`，不要把整批记录读出后再 `len(...)`。
- API Key 相关逻辑继续复用现有加密、解密、掩码工具，绝不把明文写回日志、响应或数据库。
- 混合外部调用和数据库写入时，先明确事务边界和失败恢复方式。
